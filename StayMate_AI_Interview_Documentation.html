StayMate AI — Full Stack Documentation

Author: D Jeswanth (Lead Developer) Live Site: staymate-ai-sepia.vercel.app

StayMate AI is a premium SaaS application designed to manage Paying Guest (PG) facilities and co-living operations. It automates room reservations, database logs, support ticketing, announcements, and rent status logs through custom Visitor, Tenant, and Owner portals.

1. High-Level Architecture

The application is built on a modern full-stack decoupled architecture:

  • Frontend (Client): React.js, compiled using Vite, styled with Tailwind CSS, utilizing Framer Motion for premium animations, and Redux Toolkit for state management.
  • Backend (Server): Express.js REST API using JSON Web Tokens (JWT) for secure authentication.
  • Database (Storage): MongoDB Atlas integration. In the absence of a database URL, it automatically cascades to a local file-system database written in Node using the fs module.

4. Core Coding Highlights

A. Converting Images to Base64 String (FileReader API)

To avoid cloud storage cost configurations during development, tenant complaint photos are read locally as base64 strings and stored inside the database:

const file = e.target.files[0];
const reader = new FileReader();
reader.onloadend = () => {
  setFormData({ ...formData, image: reader.result }); // Base64 Data URL
};
reader.readAsDataURL(file);

B. Role-Based Route Protection (RBAC)

API requests are verified by a custom JSON Web Token middleware inside authMiddleware.js:

const auth = (roles = []) => {
  return (req, res, next) => {
    const token = req.header('Authorization')?.replace('Bearer ', '');
    if (!token) return res.status(401).json({ msg: 'No token, authorization denied' });

    try {
      const decoded = jwt.verify(token, process.env.JWT_SECRET || 'secret');
      req.user = decoded;
      
      if (roles.length && !roles.includes(decoded.role)) {
        return res.status(403).json({ msg: 'Access denied: Insufficient permissions' });
      }
      next();
    } catch (err) {
      res.status(401).json({ msg: 'Token is not valid' });
    }
  };
};

5. Interview Preparation Q&A Cheatsheet

Q: "How did you manage frontend state?"

A: "I used Redux Toolkit. It allowed me to manage global authentication tokens and user notifications in a centralized store, preventing the need to drill props down to child elements."

Q: "What will break if the site scales to 50,000 tenants?"

A: "First, the local JSON database fallback will experience disk-write latency under parallel loads; we'd migrate strictly to MongoDB Atlas with database indexing. Second, storing images as Base64 strings will bloat our database records. We would offload image uploads to AWS S3 and store only the secure image links."

Q: "How does the room transfer transaction work?"

A: "When shifting a tenant, the backend checks for vacancy in the target room. If open, it updates the tenant record, increments occupiedBeds in the target room, and decrements it in the previous room. In production, we execute this as an atomic MongoDB database transaction."
